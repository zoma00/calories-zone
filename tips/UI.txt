No. That is only how the browser preview was built, not how the Android app works.

The file index.html was made as a simple standalone preview, so keeping the CSS inside the same file made it faster to run, easier to move around, and easier to open with a tiny local server. That choice was about convenience for the preview, not a requirement.

The actual Android app does not use HTML or CSS at all. Its UI is built in Kotlin with Jetpack Compose in CaloriesScreen.kt. In Android, styling is done through Compose theme code, components, colors, typography, spacing, and modifiers, not through separate CSS files.

So the split is:

- Browser preview: HTML plus CSS, currently kept in one file for simplicity.
- Android app: Kotlin plus Compose UI, no CSS.

If you want, I can split the preview styles into a separate CSS file now so the web preview structure looks cleaner.
